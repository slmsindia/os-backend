Online Saathi Backend
Standard Request Headers
For all JSON requests:
● Content-Type: application/json
For protected routes:
● Authorization: Bearer <JWT_TOKEN>
MODULE: AUTH
Base URL: /api/auth
POST /send-otp
Sends a 6-digit OTP to the mobile number. Rate-limited per mobile/IP (30s).
● Body: { "mobile": string }
● Response (200): { "success": true, "message": "otp sent" }
● Errors: invalid mobile, failed to send
POST /verify-otp
Verifies the OTP sent to the mobile.
● Body: { "mobile": string, "otp": string }
● Response (200): { "success": true, "message": "verified" }
● Errors: missing inputs, wrong otp, otp expired
POST /register
Registers a new user (default identity: USER).
● Body: { "mobile": string, "fullName": string, "gender": "MALE"|"FEMALE"|"OTHER",
"dateOfBirth": string, "password": password, "referredBy": userId? }
● Response (201): { "success": true, "user": { "id": string, "identity": "USER" },
"accessToken": string }
● Errors: fields missing, verify mobile first, already registered
POST /login
Authenticates a user within the tenant (resolved via domain).
● Body: { "mobile": string, "password": password }
● Response (200): { "success": true, "accessToken": string }
● Errors: credentials required, invalid mobile or pass

MODULE: USER
Base URL: /api/users Requires Authorization
GET /profile
Returns the authenticated user's profile and roles.
● Response (200): { "id", "mobile", "fullName", "identity", "roles": [{ "role": { "name":
string } }], ... }
PATCH /:id/role
Assigns a functional role (e.g., ACCOUNTANT) to a user within the hierarchy.
● Identity Access: ADMIN
● Body: { "roleName": string }
● Response (200): { "success": true, "message": "Role assigned successfully", "user": {
"id", "roles": ["ROLE_NAME"] } }
Response Standardization

json

}
